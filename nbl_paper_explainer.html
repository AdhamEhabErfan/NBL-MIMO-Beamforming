🎯 Step 1 — Problem & Motivation

Why is this paper needed?

Modern 5G and beyond-5G networks use massive MIMO — base stations equipped with dozens to hundreds of antennas — to serve many users simultaneously. To exploit those antennas, the network must steer beams toward each user. Doing this efficiently requires knowing the channel (how signals travel from BS to UE), but obtaining that channel state information (CSI) is expensive in time and bandwidth.

Core challenge: You cannot just try every possible beam direction — with hundreds of antennas and many users across many cells, the search space is enormous. You need a codebook: a pre-selected finite set of beam directions to search over.

Three interlinked problems the paper solves

Problem 1 Traditional codebooks (DFT-based) are designed for a single cell assuming a uniform user distribution, ignoring site-specific geometry.

Problem 2 Beam management links together initial access, channel sounding (CSI-RS), and data precoding. Prior work optimizes each stage in isolation — ignoring dependencies.

Problem 3 Multi-cell interference is ignored. A codebook optimal for one sector may cause heavy interference in neighbouring cells.

The paper's answer: NBL — Network Beamspace Learning. A single end-to-end neural network that jointly designs SSB and CSI-RS codebooks across all cells, trained on ray-traced site-specific channels to maximise spectral efficiency while suppressing inter-cell interference.

📡 Step 2 — MIMO & Beam Management Basics

The physical layer foundation

What is Massive MIMO?

A base station with a large antenna array (e.g., 64, 128, or 256 elements) can form narrow, high-gain beams that simultaneously serve many users on the same time/frequency resources, multiplying capacity. This is called spatial multiplexing.

Stage 1 — SSB (Synchronisation Signal Block)

The BS sweeps beams sequentially. Each UE measures the received signal power (RSRP) and reports back the index of the best beam. This narrows down the beam direction coarsely. The codebook used here is the SSB codebook.

Stage 2 — CSI-RS (Channel State Information Reference Signal)

Using the best SSB beam index as context, the BS sends a denser set of reference signals to refine the channel estimate. The CSI-RS codebook can be conditional on the SSB result. The UE feeds back a beam index (Type I) or quantized coefficients (Type II).

Stage 3 — MU-MIMO Data Transmission

The BS uses the estimated channels to compute a precoding matrix (e.g., Regularized Zero-Forcing) that simultaneously serves multiple users while suppressing inter-user interference.

📖 Step 3 — What is a Codebook?

The design variable at the heart of the paper

A codebook is a finite set of beamforming vectors (or matrices) W = {w₁, w₂, …, wₙ}. Each vector wᵢ ∈ ℂᴹ specifies the complex weights (amplitude + phase) applied to each of the M antenna elements.

Physical intuition: Each codebook entry steers a beam in a particular direction. By carefully choosing phases across the array you can reinforce signals in one direction while cancelling others — this is beamforming.

Traditional DFT Codebook

The standard approach uses the Discrete Fourier Transform (DFT) to create uniformly-spaced beams across angular space. For a Uniform Linear Array (ULA) with M elements:

# DFT codebook — N beams for M antennas wₙ = (1/√M) · [1, e^(j2π·n/N), e^(j4π·n/N), …, e^(j2π·n(M-1)/N)] # The full codebook matrix W is the DFT matrix # Pros: simple, hardware-friendly, no training needed # Cons: uniform grid ignores real site geometry & user distribution

Learned (NBL) Codebook

Instead of a fixed grid, NBL treats the codebook entries as learnable parameters of a neural network. They are updated via gradient descent to maximise spectral efficiency on real ray-traced channel data from the deployment site.

🌐 Step 4 — Network & Signal Model

Mathematical setup of the multi-cell system

System Parameters

The paper considers a multi-cell network with B base stations each having M antennas (hybrid architecture: M_RF RF chains, M antennas). Each cell serves K single-antenna UEs simultaneously. The channel is modelled over F OFDM subcarriers.

Received Signal (SSB Phase)

y_k = H_k · f_n · s + interference + noise
where H_k ∈ ℂ^(F×M) is the channel, f_n is the nth SSB codebook vector, s is the pilot symbol.

RSRP Measurement

The UE measures the Reference Signal Received Power for each codebook beam. This is the key measurement used by the neural network as its input (beamspace observations):

# RSRP for beam n, user k (averaged over subcarriers) RSRP(k, n) = (1/F) · Σ_f |H_k[f] · f_n|² # The full beamspace vector for user k: r_k = [RSRP(k,1), RSRP(k,2), …, RSRP(k,N)] # shape: (N,)

Hybrid Beamforming Constraint

In a hybrid architecture, the total precoder is factored as F = F_RF · F_BB where F_RF is an analog phase-shifter network (constant-modulus, phase-only) and F_BB is a digital baseband precoder. This constraint limits what codebook entries are physically realizable.

Key Design Insight

Steps 2–7 must all be differentiable with respect to the codebook weights for end-to-end training. Several operations (argmax for beam selection, matrix inversion for RZF) are not natively differentiable, which is why the paper introduces several mathematical tricks (see Steps 8–9).

Shared information between cells: Each BS only shares its SSB codebook (not full CSI) with neighbours. This makes the algorithm practical — the inter-cell coordination overhead is minimal yet interference is still mitigated through joint codebook design.

🔭 Step 6 — Beamspace Representation

Compressing channel information into a learnable input format

The neural network does not receive the raw channel matrix H (which would require full CSI feedback — expensive). Instead it receives the beamspace observation: the vector of RSRP measurements across all codebook beams.

Converting to Beamspace (tf_deconvert_beamspace)

The function tf_deconvert_beamspace transforms the analog beamspace RSRP measurements back into a form suitable for digital precoding. It effectively projects the beamspace observation onto the channel subspace using the codebook as a basis.

# Conceptually: # beamspace_obs shape: (batch, N_beams) # codebook W shape: (M_antennas, N_beams) # Reconstructed effective channel estimate: H_eff ≈ W · diag(beamspace_obs) · W^H # (M × M)

Why Beamspace?

Benefit 1 Dimensionality reduction: N_beams ≪ M × F (full channel size)

Benefit 2 Privacy: no raw channel shared between cells

Benefit 3 Hardware compatible: RSRP is what UEs already measure and report in 5G NR

Key paper insight: By designing the SSB codebook (which determines what the beamspace looks like) jointly with the downstream CSI-RS selection and precoding, the network learns to embed maximally useful information in the beamspace representation for its own downstream tasks.

🧠 Step 7 — NBL Neural Network Architecture

What the model looks like and what it outputs

Inputs

For each user k, the NN receives the beamspace observation vector r_k (N-dimensional RSRP measurements from the SSB sweep). For a multi-cell setup, this may include RSRP from neighbouring cells' SSB beams too.

Two-headed output

Head 1
SSB Codebook W_SSB
Shared globally across all users
Head 2
CSI-RS Codebook W_CSI
Conditionally selected per user

Architecture Details

The network uses fully-connected layers with ReLU activations. The output layer produces complex-valued codebook entries, which are then projected onto the feasible set (unit-modulus constraint for analog beamforming) using a phase-normalization operation: w = e^(j·angle(w_raw)).

Multi-cell coordination

In the network setting, each BS's NN shares a common SSB codebook as a global parameter. The CSI-RS codebook selection is local but trained with a loss that penalises inter-cell interference, encouraging the network to learn complementary beams across sectors.

Parameter efficiency: The SSB codebook has M×N_SSB complex parameters. For M=64, N=16 that is ~2,048 learnable values. This is extremely compact compared to deep channel estimation networks.

📉 Step 8 — Training & Loss Function

What the network optimises and how

Primary Loss: Negative Spectral Efficiency

ℒ = −SE_network = −(1/K) Σ_k log₂(1 + SINR_k)
The network maximises the average per-user achievable rate (Shannon capacity formula) across the entire multi-cell network.

SINR (Signal-to-Interference-plus-Noise Ratio)

SINR_k = |h_k^H · p_k|² / (σ² + Σ_{j≠k} |h_k^H · p_j|²) # h_k = effective channel for user k # p_k = precoding vector for user k (from RZF) # σ² = noise power # Interference comes from other users AND other cells

Auxiliary Beam Alignment Loss

To help convergence, an auxiliary loss encourages the selected beam to have maximum overlap with the true channel direction. This is the beam alignment accuracy metric (fraction of users whose selected beam maximises their true RSRP).

Training procedure

Step 1 Generate a batch of ray-traced channel samples H for random UE positions.

Step 2 Forward pass: compute beamspace → select CSI-RS beams → estimate channels → compute RZF precoder → compute SINR → compute SE.

Step 3 Backward pass: backpropagate through the entire differentiable pipeline to update codebook weights.

Step 4 Project codebook weights onto unit-modulus constraint after each gradient step.

⚙️ Step 9 — Differentiability Tricks

Making non-differentiable operations trainable

Problem 1: Argmax (beam selection) is not differentiable

Selecting the best beam requires an argmax over RSRP values — a discrete, non-differentiable operation. The paper uses a Straight-Through Estimator (STE):

# Forward pass: use hard argmax (discrete selection) selected = tf.one_hot(tf.argmax(rsrp, axis=-1), depth=N) # Backward pass: treat as identity (gradient passes through) @tf.custom_gradient def straight_through_argmax(x): y = tf.one_hot(tf.argmax(x, axis=-1), depth=x.shape[-1]) def grad(dy): return dy # pass gradient as-is return y, grad

This is what the notebook's TensorFlow Gradient Shim (re-implementing tf.grad_pass_through) provides in Phase 2.

Problem 2: Matrix inversion (RZF precoder) can explode gradients

Regularized Zero-Forcing requires inverting a matrix: P = H^H(HH^H + αI)⁻¹. The regularisation parameter α prevents numerical instability and also keeps gradients well-behaved during training.

Problem 3: tf.grad_pass_through removed in TF 2.16+

The notebook's compatibility patch re-implements this function using @tf.custom_gradient, which is the modern TF API for defining custom backward passes. This is purely a software compatibility fix — the math is identical.

Why this matters for the code: Cell 2 in the notebook applies all three patches before any model code runs. If you skip these patches, the training loop will crash on non-differentiable operations or raise deprecated-API errors.

📐 Step 11 — DFT & Baseline Codebooks

What NBL is benchmarked against

DFT_codebook function

The DFT codebook is the primary baseline. For a ULA with M antennas and N beams:

def DFT_codebook(M, N): # Create N uniformly-spaced beams over [-90°, +90°] angles = np.linspace(-1, 1, N) # spatial frequencies steering = np.exp(1j * np.pi * np.outer(np.arange(M), angles)) return steering / np.sqrt(M) # normalised, shape (M, N)

🧮 Step 12 — MU-MIMO & RZF Precoding

How the data transmission stage works

Multi-User MIMO (MU-MIMO)

After beam management, the BS has an estimate of each user's channel vector h_k ∈ ℂᴹ. It stacks these into a channel matrix H = [h₁, h₂, …, h_K]^H ∈ ℂ^(K×M). The goal is to transmit K independent data streams simultaneously.

Regularized Zero-Forcing (RZF)

# tf_get_RZF implementation concept: def tf_get_RZF(H, alpha): # H: (batch, K, M) complex channel matrix Hh = tf.linalg.adjoint(H) # (batch, M, K) HHh = tf.matmul(H, Hh) # (batch, K, K) reg = alpha * tf.eye(K) # regularisation P = tf.matmul(Hh, tf.linalg.inv(HHh + reg)) # Normalise columns to satisfy power constraint return P / tf.norm(P, axis=1, keepdims=True)

tf_MU_MIMO_Opt

This function wraps RZF precoding with user scheduling: it selects which K users to serve (out of all active UEs) and computes the per-user SINR and achievable rate. At evaluation time, it loops over candidate user subsets and picks the one maximising sum rate.

Why RZF and not something more complex? RZF provides a good trade-off between performance and computational tractability. It's also differentiable (matrix inversion is differentiable), making it compatible with the end-to-end training gradient flow.

Finding 1: Beam Alignment Accuracy

The DFT codebook misses the optimal beam direction for a large fraction of users — especially those near cell edges or in clustered urban environments. NBL learns to concentrate its beams where users actually are, reducing miss rate dramatically.

Finding 2: Interference Mitigation

Because NBL is trained with a multi-cell loss, it learns to steer its SSB beams away from directions that would cause strong interference to adjacent cells. DFT beams, being uniform, create high inter-cell interference at sector boundaries.

Finding 3: Transfer Learning

A codebook trained on one city/scenario can be fine-tuned to a new deployment with limited data, retaining most of its performance advantage over DFT baselines.

Finding 4: Scalability

The performance gains are larger for bigger arrays (128, 256 antennas) — the richer beamforming degrees of freedom give NBL more room to outperform the rigid DFT structure.

Summary

The paper's central contribution — NBL — is elegant: treat the codebook itself as a neural network parameter, feed it ray-traced channels, and train end-to-end with spectral efficiency as the loss. The result is a compact, site-specific codebook that outperforms decades of analytic codebook design by learning the geometry of the actual deployment.
